# Decision memo

**Conclusion:** The evidence does not support the public claim that “AI coding assistants reduce software defects by 30%.” That wording asserts a precise, causal, broadly applicable effect. Only the vendor case series reports 30%, but it has no control group, undisclosed selection criteria, and a simultaneous 55% increase in deployments, making attribution and even comparison of raw ticket counts unreliable.

The other evidence does not validate the claim. The randomized study found faster completion, but no statistically significant accuracy improvement on isolated tasks. Internal users had 18% fewer defects per release, yet they were more senior and increased automated test coverage, so the assistant’s contribution cannot be separated. Developer beliefs measure perception, not defects. The outage postmortem establishes a plausible failure mode, but not its frequency or net effect.

**Decision:** Do not use the 30% defect-reduction claim. A defensible statement is: “Some observational evidence associates assistant use with fewer defects, but a causal reduction has not been established.”

**Next evidence:** Run an adequately powered, preregistered field experiment that randomizes assistant access across comparable teams or uses a randomized phased rollout. Measure escaped defects by severity per standardized unit of software change, alongside deployments, test coverage, developer seniority, and baseline defect rates. Estimate the effect with uncertainty and independently review the analysis before making a quantified claim.
