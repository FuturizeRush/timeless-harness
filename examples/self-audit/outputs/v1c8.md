# Decision memo

**Conclusion:** The evidence does not establish that AI coding assistants cause any reduction in software defects, much less a 30% reduction.

**Decision:** Do not publicly claim, “AI coding assistants reduce software defects by 30%.” The vendor’s 30% figure is an uncontrolled before-and-after result from six undisclosed selections; deployments also changed substantially, so it cannot isolate the assistant’s effect. The internal 18% association is also inconclusive because assistant users were more senior and raised automated test coverage. The randomized study supports faster completion on isolated tasks, not fewer defects, and found no statistically significant accuracy difference. The survey measures belief, not defects. The outage postmortem shows a possible failure mode, but one incident cannot estimate overall defect impact.

The strongest supportable public statement is: “Early observational evidence suggests AI coding assistants may be associated with fewer reported defects in some settings, but a causal reduction has not been established.”

**Next evidence:** Run a prospective, adequately powered controlled rollout on representative production teams. Predefine defect rate and severity, use a stable denominator such as defects per deployment or code change, track assistant exposure, and keep seniority, testing practices, and workload comparable. Include production incidents and independently adjudicate defects. A replicated result near 30% would then support a quantified claim.
