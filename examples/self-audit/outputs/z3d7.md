# Decision memo

**Strongest defensible conclusion:** The supplied evidence does not establish that AI coding assistants reduce software defects, and it does not support a causal 30% reduction. The vendor’s 30% figure comes from an uncontrolled, selectively described case series during a period when deployment volume changed substantially. The internal 18% association is also non-causal: assistant users were more senior and expanded automated test coverage. The randomized study supports faster completion on isolated tasks, but found no statistically significant accuracy improvement. Developer perceptions are not defect measurements, while the outage postmortem demonstrates a potential failure mode without quantifying its frequency.

**Decision:** Do not make the public claim “AI coding assistants reduce software defects by 30%.” If communication is necessary, say: “Some observational evidence suggests AI coding assistants may be associated with fewer defects, but the size and cause of any reduction are not yet established.” Keep productivity claims separate from quality claims.

**Next evidence to collect:** Run a sufficiently powered, preregistered team-level randomized or stepped-wedge study in normal production work. Measure defects per deployment or change, escaped defects, and severity over a meaningful baseline and follow-up period. Track assistant usage, developer seniority, workload, deployment volume, and automated test coverage so the assistant’s contribution can be distinguished from concurrent process improvements. Use blinded defect classification and independent analysis where feasible.
